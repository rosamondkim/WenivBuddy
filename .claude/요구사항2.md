현재 만들어진 프로젝트에서 어떤 라이브러리 및 언어를 사용하는지 정리해줘
프로젝트 구조가 어떻게 이루어지고 있는지 알려주고,
초기 세팅된 해당 프로젝트에서 실행하면 나오는 에러들을 1차로 해결해줘

# 답변

## lib 폴더 파일별 역할 정리

### 1. **keyword-extractor.js**
**역할**: 로컬 기반 키워드 추출 (규칙 기반)

**주요 기능**:
- 한글 자모 분리 및 정규화
- 기술 용어 추출 (React, VSCode, JavaScript 등 대소문자 구분)
- 불용어 필터링 (은, 는, 이, 가 등)
- 2-3글자 이상의 명사 추출 (휴리스틱 방식)
- N-gram 생성 (단어 조합으로 구문 매칭)
- Jaccard 유사도 계산
- 키워드 매칭 개수 카운팅

**함수**:
- `extractKeywords(question)`: 질문에서 키워드 추출
- `calculateSimilarity(keywords1, keywords2)`: 두 키워드 집합의 유사도 계산
- `countKeywordMatches(text, keywords)`: 텍스트에 키워드 매칭 개수
- `extractKeywordsWithDetails(question)`: 디버깅용 상세 정보 추출

---

### 2. **keyword-mapping.js**
**역할**: 한글↔영문 용어 매핑 및 오타 수정

**주요 기능**:
- 한글 용어를 표준 영문 용어로 변환 (예: "리액트" → "React", "자스" → "JavaScript")
- 자주 발생하는 오타 수정 (예: "vsc" → "VSCode", "리엑트" → "React")
- 용어 표준화 (다양한 표기법을 하나로 통일)

**주요 상수**:
- `KO_TO_EN_MAPPING`: 한글 → 영문 기술 용어 사전 (약 50개)
- `TYPO_MAPPING`: 오타/유사어 → 정규 용어 사전

**함수**:
- `applyMapping(text)`: 텍스트에 한영 매핑과 오타 수정 적용
- `isMappedTerm(term)`: 매핑 사전에 있는 용어인지 확인
- `getStandardTerm(term)`: 용어의 표준 형태 반환

---

### 3. **llm-keyword-extractor.js**
**역할**: LLM 기반 키워드 추출 (Anthropic Claude API 사용)

**주요 기능**:
- Claude API를 호출하여 고급 키워드 추출
- 한영 변환, 오타 수정, 컨텍스트 이해
- 카테고리 자동 분류
- API 사용량 및 비용 추정

**함수**:
- `extractKeywordsWithLLM(question)`: LLM으로 키워드 추출 (API 호출)
- `isLLMAvailable()`: LLM 사용 가능 여부 확인
- `estimateLLMCost(question)`: 토큰 수 및 비용 예상 계산

**사용 모델**: Claude 3.5 Haiku

---

### 4. **hybrid-keyword-extractor.js**
**역할**: 하이브리드 키워드 추출 (로컬 + LLM 전략)

**주요 기능**:
- **1단계**: 로컬 추출 시도 (keyword-extractor 사용)
- **2단계**: 한영 매핑 적용 (keyword-mapping 사용)
- **3단계**: 신뢰도 계산 (기술 용어 포함 여부, 키워드 개수, 영문 비율 평가)
- **4단계**: 신뢰도가 낮으면 LLM 호출, 높으면 로컬 결과 사용
- **폴백**: LLM 실패 시 로컬 결과로 대체

**함수**:
- `extractKeywordsHybrid(question, options)`: 하이브리드 추출 (메인 함수)
- `calculateConfidence(keywords, originalQuestion)`: 신뢰도 점수 계산 (0~1)
- `extractKeywordsBatch(questions, options)`: 배치 처리
- `calculateStats(extractionResults)`: 추출 통계 계산

**파라미터**:
- `confidenceThreshold`: 신뢰도 임계값 (기본: 0.7)
- `forceLLM`: 강제 LLM 사용 여부

---

### 5. **qna-search.js**
**역할**: Q&A 데이터베이스 검색 및 유사 질문 찾기

**주요 기능**:
- 하이브리드 키워드 추출을 사용한 Q&A 검색
- 카테고리 필터링
- 유사도 기반 점수 계산 및 정렬
- 유사한 질문 추천

**점수 계산 방식**:
- 키워드 유사도: 50%
- 질문 텍스트 매칭: 30%
- 답변 텍스트 매칭: 20%

**함수**:
- `searchQnA(qnaList, searchQuery, category, maxResults)`: Q&A 검색 (메인)
- `getCategoryCounts(qnaList)`: 카테고리별 개수 계산
- `loadQnADatabase()`: Q&A 데이터베이스 로드
- `findSimilarQuestions(qnaList, currentQnaId, maxResults)`: 유사한 질문 찾기

---

### 6. **utils.js**
**역할**: Tailwind CSS 유틸리티

**주요 기능**:
- `cn(...inputs)`: 여러 클래스명을 병합하고 충돌 해결 (Tailwind Merge + clsx)

## 전체 흐름도

```
사용자 질문 입력
       ↓
[hybrid-keyword-extractor.js]
  1. 로컬 추출 (keyword-extractor.js)
  2. 한영 매핑 (keyword-mapping.js)
  3. 신뢰도 계산
  4. 낮으면 → LLM 호출 (llm-keyword-extractor.js)
       ↓
추출된 키워드
       ↓
[qna-search.js]
  1. Q&A 목록과 유사도 비교
  2. 점수 계산 및 정렬
       ↓
검색 결과 반환
```

## JSDoc 사용

모든 lib 파일에서 **JSDoc** 문법을 사용하여 함수를 문서화하고 있습니다:

- `@param {type} name - description`: 파라미터 설명
- `@returns {type} - description`: 리턴값 설명

이를 통해 TypeScript 없이도 타입 정보를 제공하고, VSCode에서 자동완성과 타입 힌트를 지원받을 수 있습니다.
